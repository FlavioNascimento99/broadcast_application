# Publisher Node.js API

Express API with TypeScript ready to connect to PostgreSQL, supporting Topics and Posts management. Database migrations are automated with Prisma.

## Features

- **TypeScript** for type-safe development
- **Express.js** server
- **PostgreSQL** database
- **Prisma ORM** for automated migrations and type-safe queries
- **RESTful API** for managing Topics and Posts
- **UUID-based IDs** for all entities
- **Automatic timestamps** for audit trail

## Quick Start

### Prerequisites

- **Node.js 18+** 
- **PostgreSQL 12+** — [See POSTGRESQL_SETUP.md](POSTGRESQL_SETUP.md) for installation guide
- **npm** or **yarn**

### 1. Install dependencies

```bash
npm install
```

### 2. Setup PostgreSQL (if not already done)

See [POSTGRESQL_SETUP.md](POSTGRESQL_SETUP.md) for:
- Installing PostgreSQL
- Creating a database and user
- Connection string format

### 3. Setup environment

Copy the example environment file:

```bash
copy .env.example .env
```

Edit `.env` with your PostgreSQL connection string:

```env
# For local PostgreSQL (default user)
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/publisher_db

# Or with custom user
DATABASE_URL=postgresql://publisher_user:your_password@localhost:5432/publisher_db
```

⚠️ **Never commit `.env` to version control. Use `.env.example` for templates only.**

### 4. Create database tables

Prisma will automatically create tables by syncing the schema:

```bash
npm run prisma:push
```

Expected output:
```
✔ Database synced, no schema changes or destructive changes detected
```

### 5. Start development server

```bash
npm run dev
```

Server starts at `http://localhost:3000`

## Scripts

```bash
# Development with auto-reload
npm run dev

# Build TypeScript to JavaScript
npm run build

# Start production server
npm start

# Prisma commands
npm run prisma:generate   # Generate Prisma Client
npm run prisma:migrate    # Create and apply migrations interactively
npm run prisma:push       # Push schema changes to database
npm run prisma:studio     # Open Prisma Studio (visual DB explorer)
```

## API Endpoints

For complete API documentation, see [API_DOCUMENTATION.md](API_DOCUMENTATION.md)

### Health & Status

- `GET /api/health` - Health check
- `GET /api/db/ping` - Database connectivity check

### Topics

- `POST /api/topics` - Create a topic
- `GET /api/topics` - List all topics
- `GET /api/topics/:id` - Get a specific topic
- `DELETE /api/topics/:id` - Delete a topic

### Posts

- `POST /api/posts` - Create a post
- `GET /api/posts` - List all posts
- `GET /api/posts/:id` - Get a specific post
- `GET /api/posts/topic/:topicId` - Get posts by topic
- `DELETE /api/posts/:id` - Delete a post

## Database Schema

The database contains two main tables:

### topics
- `id` (UUID) — Primary key
- `name` (VARCHAR) — Topic name
- `description` (TEXT) — Optional description
- `created_at` (TIMESTAMP) — Creation timestamp

### posts
- `id` (UUID) — Primary key
- `author` (VARCHAR) — Post author name
- `content` (TEXT) — Post content
- `topic_id` (UUID) — Foreign key to topics
- `created_at` (TIMESTAMP) — Creation timestamp

Posts are automatically deleted when a topic is deleted (CASCADE).

## Notes

- Users are identified by a simple string field, not a separate user table
- Posts are automatically cascaded deleted when a topic is deleted
- All IDs are UUIDs for better distribution across systems
- Prisma Client is automatically generated based on schema.prisma
